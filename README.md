EchoMessenger

# (C# 코딩) 에코메신저

## 개요
-C# 프로그래밍 학습

### -1줄소개: 사용자의 키보드 입력을 받아서 리스트박스에 출력하고 관리하는 프로그램

### -사용한플랫폼: C#, .NET Windows Forms, Visual Studio, GitHub

### -사용한컨트롤: Label, TextBox, ListBox, Button

### -사용한기술과구현한기능: 
 
 - Visual Studio를 이용하여 WinForms UI 디자인 및 컨트롤 배치
 
 - string 클래스(Trim, IsNullOrWhiteSpace)를 이용한 사용자 입력 데이터 처리
 
 - DateTime 클래스를 이용한 현재 시간 정보 구하기 및 문자열 포맷팅
 
 - ListBox의 Items 속성을 활용한 데이터 추가, 삭제 및 초기화 로직 구현

 
### -수업중에배우고사용했던클래스들관련된설명
 
- Form: 윈도우 창을 생성하고 관리하는 기본 클래스
 
- String: 문자열 데이터를 처리하며, 공백 제거 및 유효성 검사에 활용
 
- DateTime: 시스템의 날짜와 시간을 가져오는 기능을 제공
 
- MessageBox: 사용자에게 알림이나 경고창을 보여주는 정적 클래스

 ### -실습중에구현한기능들설명
 
 - 텍스트 입력 및 리스트 항목 추가 기능
 
 - 엔터키 입력 시 자동 전송 및 입력창 포커스 유지 기능
 
 - 메시지 전송 시 타임스탬프 기록 및 데이터 가공 기능
 
 - 특정 항목 선택 삭제 및 전체 내역 초기화 기능
 
 - 입력 글자 수 제한(50자) 예외 처리 기능

### -과제내용

 - Windows Forms 프레임워크를 기반으로 사용자 인터페이스를 설계하고 프로그램의 기본 컨테이너를 생성했습니다.

 - 텍스트 입력을 위한 TextBox, 결과 출력을 위한 ListBox, 동작 실행을 위한 Button 등 필수 컨트롤을 배치했습니다.

 - 사용자가 입력한 문자열 데이터를 프로그램 내부의 리스트 아이템 집합에 동적으로 연결하여 실시간 표시 로직을 구축했습니다.

 - 메시지 전송 직후 입력창의 텍스트를 자동으로 삭제하여 다음 입력을 즉시 준비할 수 있도록 상태를 관리했습니다.

### -사용한 기술과 구현한 기능

 - 인터페이스 설계: 각 컨트롤의 Name, Text, Font 속성을 정의하여 직관적인 메신저 디자인을 완성했습니다.

 - 이벤트 기반 프로그래밍: 버튼 클릭 시 특정 함수가 실행되도록 연결하여 사용자 동작에 시스템이 즉각 반응하게 했습니다.

 - 데이터 바인딩: ListBox.Items.Add() 메서드를 사용하여 런타임 중에 데이터를 실시간으로 추가하는 기능을 구현했습니다.

 - 리소스 초기화: 데이터 처리가 끝난 시점에 txtInput.Clear()를 호출하여 불필요한 잔류 데이터를 제거했습니다.

### -구현 결과 및 기능 설명

 - 사용자가 텍스트 박스에 타이핑한 후 전송 버튼을 누르면 내용이 리스트박스 하단에 새로운 줄로 즉시 추가됩니다.

 - 입력창 비우기 기능을 통해 매번 기존 글자를 지워야 하는 번거로움을 없애 실제 채팅 프로그램과 유사한 환경을 재현했습니다.

 - 데이터가 화면 범위를 초과할 경우 자동 스크롤바가 생성되어 과거 대화 내역을 누락 없이 확인할 수 있습니다.

![과제2 실행화면](img/screenshot-6.png)

![과제2 실행화면](img/screenshot-7.png)

### -과제내용
 
 - 빈 메시지나 공백만 포함된 상태에서 데이터가 전송되는 것을 방지하기 위한 유효성 검사 로직을 도입했습니다.

 - 마우스 클릭 없이 키보드 Enter 키만으로도 전송 함수를 호출할 수 있는 이벤트 가로채기 기능을 추가했습니다.

 - 전송 후 다시 입력창을 클릭할 필요가 없도록 포커스(Focus)를 자동으로 고정시키는 편의 기능을 구현했습니다.

 - 엔터키 전송 시 발생하는 윈도우 기본 경고음을 차단하여 부드러운 사용 환경을 조성하는 작업을 수행했습니다.

### -사용한 기술과 구현한 기능

 - 문자열 검증: string.IsNullOrWhiteSpace()를 사용하여 의미 없는 데이터의 전송을 원천적으로 차단했습니다.

 - 키 이벤트 매핑: KeyDown 이벤트를 감지하여 엔터키 입력 시 버튼 클릭과 동일한 로직이 실행되도록 연결했습니다.

 - 제어권 최적화: txtInput.Focus() 메서드를 활용하여 전송 후 별도의 조작 없이 다음 입력을 지원하게 했습니다.

 - 시스템 소음 제어: e.SuppressKeyPress = true 설정을 통해 불필요한 비프음이 발생하지 않도록 조치했습니다.

### -구현 결과 및 기능 설명
 
 - 조건문을 활용하여 데이터 존재 여부를 판단하며, 빈 메시지 전송 시 함수를 종료하여 데이터 낭비를 방지했습니다.

 - 마우스와 키보드 두 가지 입력 장치를 모두 지원하여 사용자의 조작 편의성과 프로그램 범용성을 확보했습니다.

 - 작업 흐름이 끊기지 않는 쾌적한 UX를 제공하며, 실제 상용 메신저 수준의 완성도 높은 인터랙션을 구현했습니다.

### -과제내용
 
 - 메시지 전송 시점의 시스템 날짜와 시간 정보를 결합하여 기록의 정밀도를 높이는 시계열 로직을 구현했습니다.

 - 입력 시 실수로 포함된 앞뒤의 불필요한 공백 문자들을 자동으로 제거하는 데이터 정제 로직을 도입했습니다.

 - 현재 리스트박스에 쌓인 전체 대화 기록의 개수를 실시간으로 계산하여 하단 라벨에 출력하는 시스템을 설계했습니다.

 - 가공된 시간 정보와 정제된 본문을 보간 기법으로 결합하여 가독성 높은 로그 형태의 데이터를 생성했습니다.


### -사용 기술 및 구현 기능

 - 시간 데이터 포맷팅: DateTime.Now를 활용하여 [HH:mm:ss] 형식의 표준 시간 문자열을 생성했습니다.

 - 데이터 클렌징: .Trim() 메서드를 적용하여 입력 스트링의 전후방 공백을 자동으로 삭제하는 기능을 구현했습니다.

 - 실시간 카운팅: Items.Count 속성을 실시간으로 추적하여 사용자에게 현재 누적 대화 수를 안내하게 했습니다.

 - 문자열 보간: $ 기호를 활용한 보간법으로 시간 정보와 메시지를 효율적이고 깔끔하게 결합했습니다.

### -구현 결과 및 기능 설명
 
 - 모든 메시지에 타임스탬프가 포함되어 기록의 신뢰성을 높였으며, 정돈된 텍스트 처리를 통해 UI 깔끔함을 유지합니다.

 - 하단 상태 바를 통해 사용자는 별도의 조작 없이도 현재 대화 건수를 실시간으로 모니터링할 수 있게 되었습니다.

 - 단순 텍스트를 고정 규격의 데이터로 변환함으로써 추후 데이터베이스 관리 및 분석에 유리한 구조를 확립했습니다.

### -과제내용
 
 - 잘못 입력된 특정 항목을 선택하여 소거할 수 있는 개별 삭제 기능을 버튼 형태로 구현했습니다.

 - 누적된 대화 내역 전체를 일괄 제거하고 시스템을 초기 상태로 되돌리는 초기화(Clear) 기능을 설계했습니다.

 - 과도하게 긴 텍스트 입력으로 인한 UI 깨짐을 방지하기 위해 최대 글자 수를 50자로 제한하는 로직을 구축했습니다.

 - 중복되는 UI 업데이트 코드를 하나의 공통 함수로 묶어 관리하는 리팩토링 작업을 통해 코드 효율성을 높였습니다.

### -사용 기술 및 구현 기능

 - 인덱스 기반 삭제: SelectedIndex를 통해 선택 항목의 위치를 파악하고 해당 데이터만 정확히 제거했습니다.

 - 컬렉션 일괄 초기화: Items.Clear() 메서드와 라벨 초기화 로직을 결합하여 데이터 상태를 리셋하는 기능을 구현했습니다.

 - 예외 처리 및 알림: 글자 수 초과 시 MessageBox.Show()를 호출하여 사용자에게 가이드라인을 제시했습니다.

 - 기능 모듈화: 반복되는 로직을 독립된 함수로 분리하여 전송, 삭제, 초기화 시점에 공통으로 재사용하게 했습니다.

### -구현 결과 및 기능 설명
 - 사용자는 개별 삭제와 전체 삭제라는 강력한 데이터 관리 권한을 갖게 되어 프로그램 제어권이 강화되었습니다.

 - 50자 제한 예외 처리를 통해 시스템 안정성을 확보했으며, 사용자에게 상황에 맞는 경고를 제공하여 신뢰도를 높였습니다.

 - 코드 중복을 제거하여 유지보수성을 최적화하였으며, 어떤 동작 시에도 하단 숫자가 정확히 동기화되도록 구현했습니다.
